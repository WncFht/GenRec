\section{七线主比较与 Fixed-Hint CE 补充}

\subsection{本章目的}

这一章对应 2026-04-11 的主比较 note，作用是把当前最常被一起讨论的几条 \texttt{Instruments-grec} RL 线放到同一个框架里，优先回答两个问题：

\begin{enumerate}
\item 如果只按 top-10 排，谁最强。
\item 如果同时看 top-10 和 coverage，哪条线最平衡。
\end{enumerate}

\subsection{统一比较口径}

\begin{itemize}
\item SFT 参考点：\texttt{checkpoint-495}
\item 主表选点：按 best \texttt{NDCG@10} 选 checkpoint
\item 覆盖峰值：另外记录 peak \texttt{HR@50}，用于识别 early spike
\item 横轴：统一成 epoch；其中 \texttt{sid-only} family 按各自完整训练长度 \texttt{2652 step = 2 epoch} 归一化，其余主线按 \texttt{3326 step = 2 epoch} 归一化
\end{itemize}

这一组图压缩出来的结论很清楚：

\begin{enumerate}
\item \texttt{rule-only rerun} 仍然是当前最强的 top-10 极值线，但 coverage 明显差，不能直接当 balanced 主线。
\item dynamic family 里，\texttt{gather-fix} 仍然是唯一长期表现稳定、并且 best 点还能把 \texttt{HR@50} 拉到 SFT 水平之上的 run。

\end{enumerate}

这一块最重要的不是“CE 一定更好”，而是：

\begin{itemize}
\item full \texttt{+CE} 更像正则项，能把轨迹变平，但 headline top-10 仍然不够强。
\item \texttt{hintce-2} 已经不再只是 early-window readout，而是一个完整轨迹上更有前景的 CE 候选。
\item 即便如此，fixed family 的主问题仍然没有变成“再堆 CE 系数”，而是如何在不损失平衡性的前提下把 clean fixed 主线继续抬高。
\end{itemize}

\subsection{本章小结}

\begin{enumerate}
\item 当前主比较应该继续保留四类 reference：plain rule-only、dynamic gather-fix、clean fixed sid-only、historical old fixed。
\item 如果目标是 coverage，不应该只看最终 checkpoint；fixed taskfix 的 early coverage spike 仍然值得单独跟。
\item 如果继续追 CE，最值得追的是 \texttt{hintce-2} 一类“中后段 balanced 优势更强”的变体，而不是简单继续堆更多命名。
\end{enumerate}
