\subsection{合并范围}

本稿当前先合并三篇和 \texttt{Instruments-grec} RL 主线最相关的实验 note：

\begin{itemize}
\item 2026-04-11：七线主比较 + fixed-hint CE 补充
\item 2026-04-16：dynamic-hint max1 消融
\item 2026-04-19：dual-task 过滤 / single-hint mixed tracking
\end{itemize}

这三篇的关系并不是平行重复，而是三层递进：

\begin{enumerate}
\item 2026-04-11 给出当前最常用的主 baseline 版图，是总览层。
\item 2026-04-16 专门回答 dynamic family 里的一个 focused ablation：把 online hint budget 从“最大深度为 3”收紧到“最大深度为 1”之后会发生什么。
\item 2026-04-19 继续沿着“任务过滤 / hint 只施加到部分任务”这条线，记录 single-hint mixed、dynamic dual-task、fixed dual-task 和 hintce-3 的新进展。
\end{enumerate}

\subsection{共享背景}

三篇 note 共享的研究背景基本一致：

\begin{itemize}
\item 数据集与任务：\texttt{Instruments-grec}
\item RL 起点：\texttt{Instruments-grec-sft-qwen4B-4-256-dsz0/checkpoint-495}
\item 常用对比维度：top-10 质量、top-50 coverage、是否依赖 hint scaffold、是否会出现 early coverage spike、是否能在完整 2 epoch 尾段维持优势
\end{itemize}

其中最重要的阅读原则有两个：

\begin{enumerate}
\item 主表默认按 best \texttt{NDCG@10} 选点，但不能只看一个 best 点，还要同时记录 peak \texttt{HR@50} 和 final/latest readout。
\item 不同 run 的完整训练 step horizon 不总是一样，因此横轴统一成 epoch 时，必须写清楚归一化口径。
\end{enumerate}

\subsection{为什么现在先转成 LaTeX}

把三篇 note 合并成 LaTeX，当前主要是为了解决两个问题：

\begin{itemize}
\item 给后续持续更新留一个更稳定的结构，而不是每次新结果出来都重新发明一篇 Markdown。
\item 让图表、表格、结论和后续动作开始形成可组合的“章”级结构，后面无论是继续扩写为长报告，还是拆成更细的实验小稿，都更容易维护。
\end{itemize}
