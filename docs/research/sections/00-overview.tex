\section{Instruments 数据集实验范围}

\subsection{本文只讨论什么}

这份整理稿只讨论 \texttt{Instruments-grec} 数据集上的 RL 实验，不把 Games、跨数据集 sweep、reward form 的完整历史演化和更早期的 exploratory runs 一起混进来。当前主线关注的是：

\begin{itemize}
\item SFT 与 plain \texttt{rule-only} baseline 的位置
\item canonical dynamic hint 与 fixed hint family 的相对位置
\item dynamic / fixed 家族内部几个关键变体的对照
\item 后续几条最值得继续追的消融与扩展：\texttt{max1}、CE scaling、single-hint mixed、dual-task
\end{itemize}

\subsection{共享实验背景}

\begin{itemize}
\item 数据集：\texttt{Instruments-grec}
\item RL 起点：\texttt{Instruments-grec-sft-qwen4B-4-256-dsz0/checkpoint-495}
\item Instruments 主线 launcher 目录：\path{hope/Qwen2_5-3B-Isntruct-qwen4B-4-256-MIMIGenRec-grec/}
\item 主读法：默认先看 best \texttt{NDCG@10}，再看对应的 \texttt{HR@50} 和必要时的 peak \texttt{HR@50}
\item 进度轴：不同 run 的完整 step horizon 不总相同，因此图里统一转成 epoch 时，需要显式写清楚归一化口径
\end{itemize}

\subsection{关于 W\&B 链接的说明}

当前本地稳定可解析的是 eval W\&B：来源是 \path{results/.wandb_eval_manifest.json} 里的 run id，并按仓库现有 W\&B 约定拼到 \path{wncfht/MIMIGenRec-Eval} 项目下。训练 W\&B 的 run id 当前没有随着同步结果一起保留下来，因此后文如果写 “W\&B”，默认指的是 eval W\&B；如果训练 W\&B 无法从本地结果解析，我会显式写出来。

\subsection{本文的默认约定}

\begin{enumerate}
\item \texttt{dynamic gather-fix} 被视为当前正确的 canonical dynamic 实现；后文里如果直接写 “dynamic”，默认指它。
\item old \texttt{fixed hint mixed-single} 只作为带 bug caveat 的历史参考线，不再把它当作方法定义本身。
\item corrected \texttt{fixed taskfix sid-only} 被当作当前更干净的 fixed 参考线。
\end{enumerate}
