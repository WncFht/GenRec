\section{Fixed Sid-Only vs Fixed Taskfix}

\subsection{区别是什么}

这两条线都属于 corrected fixed-hint family，但区别不是只改一个输出目录名：

\begin{itemize}
\item \texttt{fixed taskfix} 继续跑默认 mixed RL 数据变体，并复用当前 task-aware fixed-hint export 流程。
\item \texttt{fixed taskfix sid-only} 则切到了专门的 sid-only RL 数据变体，并额外走了一条 sid-only beam hint 分析 / 导出流程；脚本里能看到独立的 \path{ANALYSIS_RUN_NAME}、\path{ANALYZE_HINT_DEPTH=1} 和 sid-only 对应的 summary/details 路径。
\end{itemize}

\subsection{结论}

\begin{itemize}
\item \texttt{fixed taskfix sid-only} 主要把 top-10 hit 又往上推了一步，因此它是更合适的 clean fixed reference。

\item old fixed 仍然可以当历史上界参考，但不应该继续当方法定义。
\end{itemize}
